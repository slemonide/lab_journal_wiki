Let's consider the dimensions. X {\displaystyle X} , and all other "full" matrices are m × n {\displaystyle m\times n} . More interestingly, A {\displaystyle A} is m × r {\displaystyle m\times r} , and B {\displaystyle B} is n × r {\displaystyle n\times r} .

An m × n {\displaystyle m\times n} matrix: the m {\displaystyle m} rows are horizontal and the n {\displaystyle n} columns are vertical. Each element of a matrix is often denoted by a variable with two subscripts. For example, a 2 , 1 {\displaystyle a_{2,1}} represents the element at the second row and first column of the matrix.

Now, as we saw before, to properly debias this algorithm so that it stays close to Hastie et al's proof we need to compute A {\displaystyle A} row-by-row or column-by-column. As the illustration shows, since A {\displaystyle A} is m × r {\displaystyle m\times r} , we have m {\displaystyle m} rows, and r {\displaystyle r} columns. r < m {\displaystyle r<m} , hence to minimize the number of matrix inversions we have to do each iteration, we split A {\displaystyle A} and B {\displaystyle B} column-by-column. This will result in just 2 r {\displaystyle 2r} matrix inversions per iteration (TODO: check how many matrix multiplications ALS actually requires). This is worse than softImpute-ALS, but is the same as vanilla ALS.

Update step for A {\displaystyle A} then would be:

for i = 1  to  r {\displaystyle i=1{\text{ to }}r} do

A i ← X i ∗ B i ( B i T B i + λ I ) − 1 {\displaystyle A_{i}\gets X_{i}^{*}B_{i}(B_{i}^{T}B_{i}+\lambda I)^{-1}}

end for

Right now this is not at all different from softImpute-ALS. We simply split one step into several steps, so that we compute each column separately.

Let's analyze the dimensions:

[ m × 1 ] = [ m × 1 ] [ n × r ] ( . . . ) − 1 {\displaystyle [m\times 1]=[m\times 1][n\times r](...)^{-1}}

As you can see, this simply doesn't work out.

Yet another way

Hmm, will this work?

[ m × 1 ] = [ m × n ] [ n × 1 ] ( [ 1 × n ] [ n × 1 ] + [ 1 × 1 ] ) − 1 {\displaystyle [m\times 1]=[m\times n][n\times 1]([1\times n][n\times 1]+[1\times 1])^{-1}}

Well this seems to work (at least dimensionally). I guess one nice thing about this approach is that matrix inversion is especially easy. This probably will make it not work, though. Proper matrix inversion is especially important to computing the solution.

Iterating over rows

So, let's try the other way:

for i = 1  to  m {\displaystyle i=1{\text{ to }}m} do

A i ← X i ∗ B i ( B i T B i + λ I ) − 1 {\displaystyle A_{i}\gets X_{i}^{*}B_{i}(B_{i}^{T}B_{i}+\lambda I)^{-1}}

end for

Here we split A {\displaystyle A} row-by-row, rather than column by column. This actually seems more like ALS.

In this case, the dimensions are:

[ 1 × r ] = [ 1 × n ] [ n × r ] ( [ r × n ] [ n × r ] + [ r × r ] ) − 1 {\displaystyle [1\times r]=[1\times n][n\times r]([r\times n][n\times r]+[r\times r])^{-1}}

Seems to work out! Still, we didn't introduce any modifications to the algorithm at the moment: we simply made it so that each row is computed independently.

Intuition

Intuitively, we are computing row A i {\displaystyle A_{i}} from both known neighbors and unknown neighbors.

Imagine that we have a graph with A i {\displaystyle A_{i}} connecting to other vertices B i j {\displaystyle B_{i}^{j}} for j = 1 ⋯ n {\displaystyle j=1\cdots n} . These other vertices are split in two families: the ones for which we know the weight X i j {\displaystyle X_{i}^{j}} , and the ones for which we don't know it. Thus, we can label the edges by X i j {\displaystyle X_{i}^{j}} . For the unknown family, we can get an estimate of the values on the edge by multiplying A i {\displaystyle A_{i}} from the previous iteration with B i j {\displaystyle B_{i}^{j}} .

Now, at first glance it seems like there's nothing else we can do here.

But there is one thing we can do: In the formula we defined

Previous definitions
X ∗ := P Ω ( X ) + P Ω ⊥ ( A B T ) ∘ 1 K {\displaystyle X^{*}:=P_{\Omega }(X)+P_{\Omega }^{\bot }(AB^{T})\circ {\frac {1}{K}}}

and

B ^ a , j k T = P Ω a ( B T ) j k + P Ω a ⊥ ( B T ) j k 1 K a k {\displaystyle {\hat {B}}_{a,jk}^{T}=P_{\Omega _{a}}(B^{T})_{jk}+P_{\Omega _{a}^{\bot }}(B^{T})_{jk}{\frac {1}{K_{ak}}}}

So as we can see, in some sense we multiply and divide by K {\displaystyle K} . This doesn't affect the stationary points of the algorithm, but change other points. In other words, we can control the shape of the trajectory that the algorithm takes to a stationary point (or maybe even control which stationary point it converges to).

Let's re-define those starred variables so that it's more compatible with the iterative step discussed in this article.

X i ∗ := P { j | ( i , j ) ∈ Ω } ( X ) + P { j | ( i , j ) ∉ Ω } ( A B T ) ∘ 1 K i {\displaystyle X_{i}^{*}:=P_{\{j|(i,j)\in \Omega \}}(X)+P_{\{j|(i,j)\not \in \Omega \}}(AB^{T})\circ {\frac {1}{K_{i}}}}

Here we fix row and iterate over columns. We split columns into "known" and "unknown" ones. j {\displaystyle j} goes from 1 {\displaystyle 1} to n {\displaystyle n} . Finally, we take i {\displaystyle i} th row from K {\displaystyle K} matrix. This row corresponds to row X i ∗ {\displaystyle X_{i}^{*}}

B ^ i = P { ( j , r ) | ( i , j ) ∈ Ω , r  any } ( B ) + P { ( j , r ) | ( i , j ) ∉ Ω , r  any } ( B ) 1 K i {\displaystyle {\hat {B}}_{i}=P_{\{(j,r)|(i,j)\in \Omega ,r{\text{ any}}\}}(B)+P_{\{(j,r)|(i,j)\not \in \Omega ,r{\text{ any}}\}}(B){\frac {1}{K_{i}}}}

In words: we fix i {\displaystyle i} . Then we iterate over all r {\displaystyle r} , and split j {\displaystyle j} based on whether or not node A i {\displaystyle A_{i}} is connected to B j {\displaystyle B_{j}} by a known value.

We illustrate this below:

Here 'X' denotes unknown entries, and blank squares represent known entries. Note that usually in applications there are a lot more unknown entries rather than the known entries. 'I' is identity matrix. White squares are zeros. Brown rectangle with a strike through is the previous A {\displaystyle A} iterate. Here we set r = 2 {\displaystyle r=2} , and n = 3 {\displaystyle n=3} , just for the illustration.

Note that this uses a "non-sparse" version of the X ∗ {\displaystyle X^{*}} decomposition. The "sparse" version is:

X ∗ := P Ω ( X ) + ( A B T ) ∘ 1 K − P Ω ( A B T ) ∘ 1 K {\displaystyle X^{*}:=P_{\Omega }(X)+(AB^{T})\circ {\frac {1}{K}}-P_{\Omega }(AB^{T})\circ {\frac {1}{K}}}

I call these sparse if there is no Ω ⊥ {\displaystyle \Omega ^{\bot }} , and non-sparse if there is one. Sparse version is much better for actually solving large matrix completion problems (as is discussed in Hastie's paper).

So, in summary, this algorithm does indeed seems to be computationally feasible.

Now let's analyze debiasing properties.

Stationary points

For stationary points, we have that A {\displaystyle A} from the previous iteration is the same as the one in the present iteration:

So notice that debiased part cancels out -- we are free whatever we want with it, and stationary points will stay fixed.

In fact, it doesn't even have to depend on previous estimates -- we are free to choose it completely at random. This will not affect the stationary points.

Conjecture: The closer the value of the debiased part to the actual true value of unobserved matrix at those points, the better the landscape of the world around the stationary points.

On the average, what do we get?

This seems hard to compute because there is a matrix inverse. Well, perhaps we can do some computations for the matrix inverse if we do analysis of SVD on the inverted part, and then use some advanced high dimensional probability to compute the expected value.

It's likely doable, and even interesting because we would see what happens to a specific row of A {\displaystyle A} .

Perhaps it is possible to wait until we have a complete iteration, and then follow what Hastie et al. did.

Debiasing to improve convergence rate

See Paper:Hastie_2014_Matrix_Completion_and_Low-Rank_SVD_via_Fast_Alternating_Least_Squares#corollary_1. This corollary shows a dependence of convergence rate on bounds on eigenvalues of A T A {\displaystyle A^{T}A} and B T B {\displaystyle B^{T}B} . Of course, there these are global A {\displaystyle A} and B {\displaystyle B} that are the same for all entries. But perhaps these can be restricted just to a single row of A {\displaystyle A} like in our discussion above. So then, if we can somehow improve the bounds on eigenvalues through debiasing, we will successfully improve the convergence rate.

I think that intuitively, we want the bounds to be as tight as possible. In other words, we want these matrices to be as close as possible to identity matrices times a constant. I.e. simple scalar multiplication operators.

Rank 1

Just as an example consider rank=1 case:

We can explicitly define the matrix to be

a x b

Then, the product is ( a 2 + b 2 ) + x 2 {\displaystyle (a^{2}+b^{2})+x^{2}} .

The debiased matrix would be:

a x λ {\displaystyle x\lambda } b

And the product would be ( a 2 + b 2 ) + x 2 λ 2 {\displaystyle (a^{2}+b^{2})+x^{2}\lambda ^{2}} .

Then:

U 1 ≥ ( a 2 + b 2 ) + x 2 ≥ L 1 {\displaystyle U_{1}\geq (a^{2}+b^{2})+x^{2}\geq L_{1}}

and

U 2 ≥ ( a 2 + b 2 ) + x 2 λ 2 ≥ L 2 {\displaystyle U_{2}\geq (a^{2}+b^{2})+x^{2}\lambda ^{2}\geq L_{2}} .

As you can see, in this case we are dealing with scalars, so U 1 = L 1 {\displaystyle U_{1}=L_{1}} and U 2 = L 2 {\displaystyle U_{2}=L_{2}} . Hence, the distribution is as tight as possible.

But here we can still control the size of λ {\displaystyle \lambda } . But it becomes an optimization problem because U ≥ L {\displaystyle U\geq L} and some convergence metrics in Corollary 1 are bounded by the U {\displaystyle U} . Hence, there is an optimum value.

Modification of Corollary 1 for debiased algorithm

First, we would have to modify the bounds since in our case, B {\displaystyle B} and A {\displaystyle A} will be specific for each row of A {\displaystyle A} and B {\displaystyle B} correspondingly.

So suppose that in addition to: ℓ U I ⪰ B T B ⪰ ℓ L I {\displaystyle \ell ^{U}\mathbf {I} \succeq B^{T}B\succeq \ell ^{L}\mathbf {I} } and ℓ U I ⪰ A T A ⪰ ℓ L I {\displaystyle \ell ^{U}\mathbf {I} \succeq A^{T}A\succeq \ell ^{L}\mathbf {I} } . Where ℓ U {\displaystyle \ell ^{U}} and ℓ L {\displaystyle \ell ^{L}} are constants independent of k {\displaystyle k} .

We also have that this is true for all specific debiased versions of A and B for all rows of B and A.
