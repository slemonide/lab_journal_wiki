Things we want from this parametrization:

  • It should fit as perfectly as possible
  • The fit is a function of eddy mixing rate and partitioning scale
  • We want eddy mixing rate and partitioning scale to vary as smoothly as possible due to physical considerations

Taking these statements into account, we can express the task of computing eddy mixing rate and partitioning scale as an optimization problem.

First, fix a position ( x , y ) {\displaystyle (x,y)} . We will denote it by index i j {\displaystyle ij} since we will be discretizing things later anyway. At fixed spatial location, we have:

F i j ( σ , κ ) = 1 2 ( L i j ( σ i j ) − κ i j H i j ( σ i j ) ) 2 + λ 2 [ ( ( σ i + 1 , j − σ i − 1 , j ) 1 2 d ) 2 + ( ( σ i , j + 1 − σ i , j − 1 ) 1 2 d ) 2 + ( ( κ i + 1 , j − κ i − 1 , j ) 1 2 d ) 2 + ( ( κ i , j + 1 − κ i , j − 1 ) 1 2 d ) 2 ] {\displaystyle F_{ij}(\sigma ,\kappa )={\frac {1}{2}}(L_{ij}(\sigma _{ij})-\kappa _{ij}H_{ij}(\sigma _{ij}))^{2}+{\frac {\lambda }{2}}[((\sigma _{i+1,j}-\sigma _{i-1,j}){\frac {1}{2d}})^{2}+((\sigma _{i,j+1}-\sigma _{i,j-1}){\frac {1}{2d}})^{2}+((\kappa _{i+1,j}-\kappa _{i-1,j}){\frac {1}{2d}})^{2}+((\kappa _{i,j+1}-\kappa _{i,j-1}){\frac {1}{2d}})^{2}]}

Where d {\displaystyle d} is the cell size we have in our discretization.

Note: We can also introduce a constraint that we want things to be smooth in time as well. This will add a third index, representing time. This should improve the fit error.

Now, the function we actually want to minimize then is ∑ i j F i j {\displaystyle \sum _{ij}F_{ij}} . We minimize it over all σ a b {\displaystyle \sigma _{ab}} and κ a b {\displaystyle \kappa _{ab}} .

Alternative formulation of smoothness

This seems complicated. Luckily, there is an alternative formulation of smoothness. Say we pick a small region in a filed. Something like 11x11 cells. We then pick a constant partitioning scale and eddy mixing rate that fits all these cells the best, and assign this value to the central cell in the region.

This could lead to trouble, though, if all these cells are very similar. Also, with this approach we might be loosing smaller scale variations in partitioning scale and eddy mixing rate. But this is somewhat similar to the λ {\displaystyle \lambda } regularization parameter in the previous formulation.

With this approach, we would have say 11x11 square of 121 measurements, and then we can compute a linear fit for eddy mixing rate for each partitioning scale. We would then pick the optimal eddy mixing rate and partitioning scale. It would be interesting to visualize how well the parametrization works for different values in general. For this visualization we could plot partitioning scale on one axis, and eddy mixing rate on the other one.

To improve localization of small-scale features of parameters, we could assign importance weights to entries: The further they are from the center, the less important they are. It would be a weighted linear fit for eddy mixing rate.

How could we assign the importance? Well, the decay rate would depend on the partitioning scale. We could probably use the same kernel we use to partition the field to also assign the importance. It can also probably be used to get the correct size of the fitting region (this will actually prevent large partitioning scales be chosen plainly for the reason that they give smoother fields). We should also encourage larger fields as they could give better overall fit (use chi-squred? or just divide the fitness by the number of variables?).
